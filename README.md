# PigeonPiper
A Professional Log Transport Tool 一个高效日志下载打包工具✧基于C++&amp;Qt

**Pigeonpiper** 是一款基于 Qt 开发的高性能、可视化文件传输与管理工具。它底层深度集成 `rsync`，旨在为 Linux 环境（特别是 RHEL 7 等企业级旧版本系统）提供稳定、精准且具备全局进度反馈的文件同步体验。

# ✨ 核心特性
- 智能进度追踪：完美支持 rsync 3.1.0+ 的 --info=progress2 参数，提供全局任务进度条，告别单个文件循环跳动。
- 极致环境兼容：
  - 支持自定义 rsync 执行路径。
  - 内置版本探测逻辑：如果系统 rsync 过旧（如 RHEL 7 默认版本），可无缝切换至自定义rsync。
- 多服务器管理：通过侧边栏树状结构快速切换不同的远程服务器及预设路径。
- 自动化归档：内置 tar 压缩支持，可根据自定义规则（后缀名）自动处理特定文件。
- 多语言支持：原生支持简体中文与英文切换。
- 绿色部署：采用库文件隔离技术（libs/ 目录），通过启动脚本实现优先系统库、缺失补全的加载策略。

# 🚀 快速开始
1. 部署与启动

解压后，无需安装，直接通过启动脚本运行：
```
chmod +x start.sh Pigeonpiper
./start.sh
```
2. 配置自定义 rsync (推荐)

如果你的系统 rsync 版本低于 3.1.0，建议在“设置”窗口中指定自定义路径：
  1. 打开 Settings -> Global Configuration。
  2. 在 Custom rsync path 中输入自带 rsync 的绝对路径。
  3. 点击旁边的 Check Env Dependencies 验证版本。

# 🛠️ 环境依赖检查
Pigeonpiper 对底层工具有如下要求：
|工具|最低版本|说明|
|---|---|---|
|rsync|3.1.0|建议 3.1.0+ 以获得最佳全局进度显示效果|
|tar|任意版本|建议支持xz 用于执行文件归档操作|
|SSH|任意版本|确保已配置免密登录（SSH Key）以避免传输中断|

# ⚠️ 常见问题排查 (Troubleshooting)
- 无法显示远程文件列表？
  请检查远程主机是否安装了 rsync，且当前机器是否可以通过 SSH 无密码访问目标主机。
- 进度条不动或跳变？
  请在设置中检查 rsync 版本。版本低于 3.1.0 时不支持全局进度计算，程序会自动回退到单文件进度模式。
- 启动报错：libQt5Core.so.5: not found？
  请确保通过 ./start.sh 而不是直接双击二进制文件启动，脚本会自动将 libs/ 加入加载路径。
